Unable to delete a non session-scoped locked node in XA Environment
You must first add a valid lockToken to the Session and then try to remove this node in a XA Environment. This will resulting in a NoSuchItemStateException State has been marked destroyed. The problem is that the unlock Operation will be done after that the node has been marked for destroyed. Patch applied in rev 820925Merged to the 1.6 branch in revision 897920.
